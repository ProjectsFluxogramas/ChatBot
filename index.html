Instalação Local — Totem
  • Programa Python instalado diretamente no totem — executa 100% local
  • Python via PortalCompany — instalação homologada

  • App desktop nativo
  • Usuários que vão usar a ferramenta: Blue collar (não possuem e-mail corporativo)
🔐
Autenticação (RBAC)
  • Interface gráfica com tela de cadastro login e senha
  • Sugestão de senha forte no cadastro de novos usuários
  • ≥ 8 caracteres • A–Z • 0–9 • caractere especial (!@#$…)
  • Senhas armazenadas com hash SHA-256
  • Perfis: Operador e BPE
🗄️
Auditoria (SQLite — banco local no totem)
  • Banco de dados SQLite armazenado no próprio totem — não trafega pela rede
  • Todo acesso e toda pergunta registrados localmente
  • Rastreabilidade completa: quem, o quê, quando...
📂
Documentos da Operação (ITs)
  • Documentos .txt carregados de pasta local — nenhum arquivo é enviado externamente
  • Processamento via RAG em Python: geração de embeddings e índice vetorial local
  • Durante a consulta, o sistema recupera apenas chunks semanticamente relevantes
  • Somente esses trechos são enviados para a IA para geração da resposta
⚠️  Único dado enviado para a IA:  pergunta do usuário + trechos relevantes das ITs (chunks) usados como contexto para gerar a resposta  →  requisição HTTPS para API Gemini (Google) com faturamento ativo e política de não retenção de dados
🌐 Externo — Google Gemini API (HTTPS outbound)
🤖
Gemini 2.5 Flash — Geração de Resposta
  • Recebe: pergunta + trechos das ITs + histórico recente
  • Não recebe: documentos completos, credenciais, logs ou dados de sistema
  • System prompt: responder somente com base no contexto fornecido

Chatbot CEVA & Logitech — Visão Gerencial

Como a solução funciona · Para gestores e time de TI · Sem jargão técnico

📶
Conectividade de rede
O totem usa o WiFi corporativo Ceva-Corporate para se comunicar com a IA
Todo o processamento interno da ferramenta — leitura de documentos, busca nos textos, login de usuários e salvamento do histórico — acontece dentro da própria máquina, sem precisar da rede. A conexão WiFi é usada exclusivamente no momento de enviar a pergunta ao Gemini (Google) e receber a resposta
🔒 Nenhuma informação trafega pela rede corporativa além do envio controlado ao Gemini. A rede WiFi Ceva-Corporate é o único caminho de saída de dados da aplicação.
🖥️
Arquitetura de implantação
Solução instalada localmente — sem dependência de servidores externos ou infraestrutura de nuvem própria
A ferramenta é executada diretamente no computador do totem, operando de forma autônoma. Não requer servidor dedicado. Todo o processamento — leitura de documentos, geração do índice de busca, autenticação de usuários e armazenamento de histórico — ocorre dentro da própria máquina.
📁 Todos os ativos da aplicação (documentos, banco de dados, índice de busca e credenciais criptografadas) residem em uma pasta dedicada no sistema operacional do totem — garantindo rastreabilidade e controle total pelo time de TI.
📋
Base de conhecimento operacional
O chatbot é alimentado exclusivamente pelas Instruções de Trabalho (ITs) oficiais — sem conteúdo externo ou gerado livremente pela IA
As ITs são exportadas do Intelex e carregadas na ferramenta pelo administrador responsável. A partir disso, o sistema constrói um índice de busca localmente sobre esses documentos — capaz de localizar informações por contexto e intenção, não apenas por palavras-chave exatas. Isso garante que todas as respostas sejam rastreáveis a um documento oficial.
🔄 A atualização da base é sob demanda: quando uma IT for revisada no Intelex, basta substituir o arquivo e recarregar a pasta. O sistema reprocessa automaticamente apenas os documentos alterados.
🔬
Mecanismo de recuperação e geração de respostas
A IA não responde com base em conhecimento genérico — ela é instruída a responder somente com o que está nas ITs carregadas
Ao receber uma pergunta, o sistema realiza uma busca inteligente sobre todos os documentos e seleciona apenas os trechos mais relevantes ao contexto da consulta. Esses trechos — e somente eles — são enviados ao modelo de linguagem (Gemini, do Google) junto à pergunta do usuário, com instrução explícita de responder exclusivamente com base no material fornecido.
✅ Esse mecanismo elimina o risco de respostas inventadas pela IA — garantindo que toda informação entregue ao operador seja auditável e rastreada a uma fonte oficial.

2
🗂️

3
📤
Envio controlado à IA
Somente os trechos selecionados + a pergunta são transmitidos ao Gemini via WiFi Ceva-Corporate — nenhum documento completo trafega
4
🤖
Síntese pela IA
O Gemini estrutura uma resposta clara e objetiva, circunscrita ao contexto fornecido — sem extrapolação ou conteúdo não referenciado
5
✅
Resposta rastreável
O operador recebe a resposta com indicação das fontes documentais utilizadas, preservando a cadeia de rastreabilidade
🤖
Governança sobre o uso da Inteligência Artificial
O Gemini atua como motor de linguagem — não como repositório de dados ou tomador de decisão
O modelo de linguagem do Google é utilizado exclusivamente para a etapa de síntese e formatação da resposta. Ele não recebe a base documental completa, não tem acesso ao banco de usuários, não armazena histórico de conversas e não é consultado para nenhuma finalidade além de redigir a resposta a partir do contexto previamente filtrado pelo sistema local. A conta Google utilizada opera sob política de não retenção de dados.
🔐 O controle do que é enviado à IA é exercido integralmente pelo sistema local. A transmissão ocorre via WiFi corporativo Ceva-Corporate usando protocolo HTTPS.
🛡️
Síntese de conformidade e soberania dos dados
Dados que permanecem exclusivamente no ambiente local: base documental completa (ITs), banco de dados de usuários, histórico de conversas, índice de busca e credenciais de acesso (armazenadas com hash criptográfico SHA-256).

Dados transmitidos externamente: unicamente a consulta do usuário somada aos trechos documentais selecionados, transmitidos via WiFi Ceva-Corporate → protocolo HTTPS para a API do Google Gemini.

A arquitetura foi deliberadamente desenhada para minimizar a superfície de exposição de dados, mantendo soberania operacional sobre a base de conhecimento e conformidade com políticas de privacidade corporativa.
